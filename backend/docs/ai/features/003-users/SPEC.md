# SPEC.md
# Фича: users

## Цель
Дать MVP-контракт для работы с текущим профилем пользователя и поиска активных пользователей по email.

## Текущий backend-контракт
- `GET /users/me`
- `PATCH /users/me`
- `GET /users/search?email=...`

## Бизнес-правила
- профиль доступен только аутентифицированному пользователю;
- обновление профиля не работает для `inactive` или `soft-deleted` пользователя;
- email нормализуется и должен оставаться уникальным;
- поиск возвращает только `active` и `non-deleted` пользователей;
- поиск ограничен `10` результатами.

## Что уже закрыто
- профиль читается и обновляется через `users` модуль;
- search использует trim-aware query DTO;
- есть unit и e2e покрытие базовых сценариев.

## Ограничения MVP
- нет админского управления пользователями;
- нет смены пароля и avatar workflow внутри `users`;
- нет сложных фильтров кроме email-фрагмента.

# АМЛ-таск
## О проекте

АМЛ-таск — это внутренний инструмент для управления задачами и проектами, объединяющий структурированный подход Jira (проекты, бэклог, спринты) и гибкость Trello (Kanban-доска с drag-n-drop).

Проект создается как удобное и интуитивное решение для команд, которым нужна прозрачность процессов без излишней сложности.

## Цель

Создать простой и быстрый инструмент, позволяющий:

- управлять задачами в рамках проектов
- планировать работу через бэклог и спринты
- отслеживать прогресс на Kanban-доске
- минимизировать порог входа для пользователей

## Флоу разработки

Проект разрабатывается в [GitFlow](https://habr.com/ru/articles/767424/) - возможны расхождения и иные согласованности. 

Каждый член команды работает внутри отдельной ветки. Одна ветка отвечает за реализацию отдельной фичи или же модуля. 

Примеры веток: 
``` 
backend/feature/auth
backend/ 
frontend/feature/main-page
develop/
database/ 
```

Наименование ветки зависит от разработчика, но для грамотного и чистого контроля версий желательно вынести разработку над отдельными модулями в отдельные ветки - backend, frontend, database. 

Коммиты желательно (но не обязательно!) подписывать в конвенции GitFlow. 

Пример:
```
feat: implement health get method
fix: fixed red test 
chore: upd gitignore
docs: add README.md
feat(auth): create auth screen
```

Где:

- feat — новая функциональность
- fix — исправление бага
- refactor — рефакторинг без изменения поведения
- style — изменения форматирования/стилей без изменения логики
- docs — изменения документации
- test — тесты
- chore — служебные изменения, конфиги, зависимости, скрипты


После завершения работы над модулем или фичей необходимо выполнить Pull Request в main ветку проекта. (из-за разграничения отдельных модулей на папки конфликтов возникнуть не должно).

Периодически необходимо выполнять git pull для получения "свежих" версий проекта. 

Текущие задачи на проект в данный момент времени определяет для себя каждый из разработчиков - индивидуально. Вести беклог задач можно внутри проекта в [GitHub Issues](https://github.com/risamey/aml-task/issues) или же любым другим удобным способом. 

Позже для общего видения необходимо будет создать любую доступную kanban-доску для ведения данного проекта.
